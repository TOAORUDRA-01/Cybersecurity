LINUX ARCHITECTURE

How the Linux OS is structured — from hardware up to user applications. Understanding this makes everything else click.

Kernel — The Core

Manages hardware, memory, processes, and syscalls. Runs in privileged "ring 0". Users never interact with it directly — they go through the shell or system calls.

  • Process scheduling (who runs when)
  • Memory management (virtual address spaces)
  • VFS (unified file access layer)
  • Network stack (TCP/IP implemented here)

Shell — Your Interface

The shell (bash, zsh, sh) is a program that takes your text commands, interprets them, and calls the kernel on your behalf via system calls.

FILESYSTEM HIERARCHY

The FHS (Filesystem Hierarchy Standard) — what lives where and why. Navigate with confidence, find config files, and know where secrets hide.

/ ← root of everything ├── bin/ ← essential user binaries: ls, cp, cat, bash ├── sbin/ ← system binaries (root): ifconfig, fdisk, iptables ├── etc/ ← ALL system configuration files │ ├── passwd ← user accounts (world-readable!) │ ├── shadow ← password hashes (root only) │ ├── sudoers ← sudo permissions │ ├── hosts ← static DNS (IP→hostname mappings) │ ├── ssh/ ← SSH server config │ └── crontab ← system-wide scheduled tasks ├── home/ ← user home directories │ └── rudra/ ← your home: ~/ │ ├── .ssh/ ← SSH keys (600 permissions!) │ ├── .bashrc← bash config, aliases, PATH │ └── .bash_history ← command history ├── proc/ ← virtual FS: running processes, kernel info ├── sys/ ← virtual FS: hardware/kernel parameters ├── var/ ← variable data │ ├── log/ ← system logs: auth.log, syslog, kern.log │ └── www/ ← web server files (Apache/Nginx) ├── tmp/ ← temp files, world-writable → pentest hotspot ├── usr/ ← user programs │ ├── bin/ ← most user commands (python, gcc, nmap) │ └── local/ ← manually installed software ├── dev/ ← device files (disks, null, random) ├── mnt/ ← mount points for external drives └── root/ ← root user's home directory

find — The Most Powerful Search Tool

security
▾

find searches the filesystem recursively with incredibly flexible criteria. Essential for privilege escalation and forensics.

bash
# Basic syntax: find [path] [criteria] [action]

# ── BY NAME ─────────────────────────────────
find / -name "flag.txt"            # exact name
find / -name "*.conf"              # wildcard
find / -iname "secret*"            # case-insensitive

# ── BY TYPE ─────────────────────────────────
find /home -type f                  # files only
find /home -type d                  # directories only
find /dev -type l                   # symlinks

# ── BY PERMISSIONS (PRIVESC GOLD!) ──────────
find / -perm -u=s -type f 2>/dev/null  # SUID files → privesc!
find / -perm -g=s -type f 2>/dev/null  # SGID files
find / -perm -777 -type f 2>/dev/null  # world-writable executables
find / -writable -type d 2>/dev/null    # writable directories

# ── BY OWNER ────────────────────────────────
find / -user root                    # owned by root
find / -nouser                       # orphan files (no owner)

# ── BY TIME ─────────────────────────────────
find / -mtime -7                     # modified in last 7 days
find / -mmin -60                     # modified in last 60 minutes
find / -newer /tmp/ref               # newer than reference file

# ── WITH ACTIONS ────────────────────────────
find /tmp -name "*.sh" -exec chmod +x {} \;  # exec command on results
find /var/log -name "*.log" -delete           # delete matching files
find / -name "*.bak" -ls                      # list details of matches
PrivEsc Tip
find / -perm -u=s -type f 2>/dev/null — Run this on every box you get access to. SUID binaries running as root are prime privilege escalation targets. Cross-reference results with GTFOBins.

PERMISSIONS & OWNERSHIP

Linux's permission system controls who can read, write, or execute every file. Misconfigurations here are the #1 source of privilege escalation.

Every file has three permission sets: Owner (u), Group (g), and Others (o). Each set has three bits: Read (r=4), Write (w=2), Execute (x=1).

chmod, chown — Changing Permissions

commands
▾
bash
# ── CHMOD (change mode) ─────────────────────
chmod 755 script.sh         # octal: rwxr-xr-x
chmod 600 ~/.ssh/id_rsa     # SSH key: owner rw only
chmod +x script.sh          # add execute for all
chmod u+x script.sh         # add execute for owner only
chmod o-r secret.txt        # remove read from others
chmod g+w shared.txt        # add write for group
chmod -R 755 /var/www/      # recursive

# ── CHOWN (change owner) ─────────────────────
chown root file.txt          # change owner to root
chown rudra:developers file  # owner:group
chown -R www-data /var/www/  # recursive chown

# ── SPECIAL BITS ─────────────────────────────
chmod u+s /usr/bin/program   # set SUID — runs as file owner
chmod g+s /shared/dir/       # set SGID — new files inherit group
chmod +t /tmp/               # sticky bit — only owner can delete

# In ls -l, special bits show as:
# -rwsr-xr-x  SUID set (s in owner execute position)
# drwxrwsr-x  SGID set (s in group execute position)
# drwxrwxrwt  Sticky bit (t in others execute position)

# ── UMASK (default permissions) ──────────────
umask                         # show current (e.g. 0022)
umask 027                    # new files: 640, dirs: 750
SUID Exploitation
When a binary has the SUID bit and is owned by root, it runs as root regardless of who executes it. If that binary can be abused to execute arbitrary code (e.g., via shell escape), you get root. Common SUID abuse: find, vim, less, bash, python.

User & Group Management

commands
▾
bash
# ── USER INFO ────────────────────────────────
whoami                     # current username
id                         # uid, gid, groups — essential first command
id rudra                   # info about specific user
who                        # who is logged in

# ── CREATE / MODIFY USERS ────────────────────
sudo useradd -m -s /bin/bash newuser  # create user with home + bash
sudo passwd newuser         # set password

sudo userdel -r olduser    # delete user + home dir
sudo chsh -s /bin/zsh rudra # change login shell

# ── GROUPS ───────────────────────────────────
groups                     # show your groups

sudo usermod -aG hackers rudra  # add user to group

# ── SUDO ─────────────────────────────────────
sudo command               # run as root
sudo -l                    # list what THIS user can sudo — CRITICAL ENUM!

sudo -s                    # root shell via sudo
su root                    # switch to root (need root password)
su - rudra                 # switch user (full login shell)

# ── PRIVESC CHECK ────────────────────────────
cat /etc/passwd | grep -v nologin  # users with real shells
cat /etc/sudoers                    # who can sudo what
sudo -l                              # ← run this every time!
Enum Tip
id then sudo -l are the first two commands to run when you get a shell. They instantly reveal privilege opportunities. If you see (ALL) NOPASSWD: ALL in sudo -l output, type sudo bash and you're root.

TEXT PROCESSING

grep, awk, sed, cut, sort — the Unix pipeline tools that make data manipulation trivially powerful.

The Unix Philosophy
Write programs that do one thing well. Write programs that work together. Write programs that handle text streams. The pipe (|) connects them — cmd1 | cmd2 | cmd3 — output of one becomes input of the next.

grep — Search Text

security essential
▾
bash
# ── BASIC GREP ────────────────────────────────
grep "root" /etc/passwd           # search for pattern in file
grep -i "password" config.txt    # case-insensitive
grep -r "secret" /var/www/       # recursive search in dir
grep -n "error" app.log          # show line numbers
grep -v "nologin" /etc/passwd    # invert match (exclude)
grep -c "failed" auth.log        # count matching lines
grep -l "password" /etc/*.conf   # list filenames only
grep -A 3 "error" log.txt        # 3 lines After match
grep -B 3 "error" log.txt        # 3 lines Before match
grep -C 3 "error" log.txt        # 3 lines Context (before + after)

# ── REGEX PATTERNS ────────────────────────────
grep -E "^root" /etc/passwd      # ^ = start of line
grep -E "bash$" /etc/passwd      # $ = end of line
grep -E "[0-9]{1,3}\.[0-9]{1,3}" log # IP-ish pattern
grep -oE "[a-zA-Z0-9._%+-]+@[a-zA-Z0-9.-]+" file  # extract emails
grep -oE "[0-9]+\.[0-9]+\.[0-9]+\.[0-9]+" log   # extract IPs

# ── SECURITY HUNTING ──────────────────────────
grep -rn "password\|passwd\|secret\|key" /etc/ 2>/dev/null
grep -r "PRIVATE KEY" /home/ 2>/dev/null       # find SSH keys
grep -rn "DB_PASS\|db_password" /var/www/       # DB creds in web files

awk, sed, cut — Data Extraction & Transformation

power tools
▾
bash
# ── AWK — column/field processor ─────────────
# Fields split by whitespace: $1=1st, $2=2nd, $NF=last
awk '{print $1}' /etc/passwd    # print 1st field
awk -F: '{print $1}' /etc/passwd # use : as delimiter
awk -F: '{print $1,$3}' /etc/passwd  # username and UID
awk -F: '$3==0 {print $1}' /etc/passwd  # users with UID 0 (root!)
awk '{sum+=$1} END {print sum}' numbers.txt # sum column
awk '/error/{count++} END{print count}' app.log # count pattern

sed -i 's/old/new/g' file.txt  # in-place edit

sed '/^$/d' file.txt            # delete empty lines
sed '1,5d' file.txt            # delete lines 1-5
sed -n '10,20p' file.txt       # print lines 10-20

# ── CUT — extract columns ─────────────────────

# Top 10 IPs in access log by request count:

BASH SCRIPTING

Automating tasks, writing recon scripts, and understanding scripts you encounter during pentests.

Script Anatomy & Variables

scripting

#!/bin/bash         # shebang — tells OS which interpreter to use
# Script: recon.sh
# Usage: ./recon.sh 192.168.1.5

# ── VARIABLES ────────────────────────────────
TARGET="$1"          # $1 = first argument, $2 = second, etc.
PORT=80
NAME="rudra"
DATE=$(date +%Y%m%d)  # command substitution
RESULT=$(ping -c1 $TARGET 2>&1)

# ── SPECIAL VARIABLES ─────────────────────────
# $0   = script name
# $1-9 = positional arguments
# $@   = all arguments as array
# $#   = number of arguments
# $?   = exit status of last command (0=success)
# $$   = current PID
# $!   = PID of last background process

# ── ARRAYS ────────────────────────────────────
PORTS=(22 80 443 8080)
echo ${PORTS[0]}        # first element: 22
echo ${PORTS[@]}        # all elements
echo ${#PORTS[@]}       # array length

Control Flow — if, for, while, case

scripting
▾
bash
# ── IF / ELIF / ELSE ─────────────────────────
if [ $# -eq 0 ]; then
    echo "Usage: $0 <target>"; exit 1
elif [ $# -eq 1 ]; then
    echo "One argument: $1"
else
    echo "Multiple arguments"
fi

# Common test conditions:
# [ -f file ]  = file exists and is regular file
# [ -d dir ]   = directory exists
# [ -r file ]  = file is readable
# [ -z "$v" ]  = variable is empty
# [ "$a" == "$b" ] = string equality
# [ $n -gt 10 ]    = numeric: greater than
# [ $n -eq 0 ]     = numeric: equal

# ── FOR LOOP ──────────────────────────────────
for port in 22 80 443 8080; do
    nc -zv $TARGET $port 2>&1 && echo "OPEN: $port"
done

# Loop over range:
for i in {1..254}; do
    ping -c1 -W1 "192.168.1.$i" >/dev/null && echo "UP: 192.168.1.$i"
done

# Loop over lines in file:
while IFS= read -r line; do
    echo "Processing: $line"
done < wordlist.txt

# ── WHILE LOOP ────────────────────────────────
count=0
while [ $count -lt 5 ]; do
    echo "Count: $count"
    (( count++ ))
done

# ── CASE ──────────────────────────────────────
case $1 in
    scan)   nmap -sV $2 ;;
    brute)  hydra $2 -l admin -P rockyou.txt ssh ;;
    *)      echo "Unknown: $1" ;;
esac

#!/bin/bash
# Simple recon script — safe, for your own systems

RED='\033[0;31m'; GREEN='\033[0;32m'; AMBER='\033[0;33m'; NC='\033[0m'

banner() {
    echo -e 
    echo -e 
    echo -e 
    echo -e 
    echo -e 
}

check_arg() {
    if [ $# -eq 0 ]; then
        echo -e "${RED}Usage: $0 <target_ip>${NC}"; exit 1
    fi
}

ping_check() {
    echo "[*] Checking if host is alive..."
    if ping -c1 -W2 $TARGET >/dev/null 2>&1; then
        echo -e "${GREEN}[+] Host is UP${NC}"
    else
        echo -e "${RED}[-] Host may be DOWN (or blocking ICMP)${NC}"
    fi
}

port_scan() {
    echo "\n[*] Scanning common ports..."
    for port in 21 22 23 25 53 80 110 443 445 3306 3389 8080; do
        timeout 1 bash -c "echo >/dev/tcp/$TARGET/$port" 2>/dev/null && \
            echo -e "${GREEN}  [OPEN]${NC} $port" || \
            echo -e "${RED}  [----]${NC} $port"
    done
}

dns_lookup() {
    echo "\n[*] DNS lookup..."
    HOST=$(host $TARGET 2>/dev/null | awk '{print $NF}')
    echo "    Hostname: ${HOST:-unknown}"

PrivEsc — Cron Path Hijack
If a cron job runs /opt/scripts/backup.sh and that file is world-writable, you can overwrite it with a reverse shell. Also check if cron uses relative paths (PATH hijack): cron runs backup without full path, and your writable $PATH directory comes first.

grep "Accepted password" /var/log/auth.log      # successful logins

# Count failed login attempts per IP:
grep "Failed password" /var/log/auth.log | awk '{print $11}' | sort | uniq -c | sort -rn | head -10

# Top IPs by request count:
